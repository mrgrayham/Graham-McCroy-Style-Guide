Graham McCroy Style Guide

This guide & code repository is used for designing the human resource management system at various organizations.

Buttons

A button is a UI element that requires action by the user to perform a function.

Each button will have a default size but color varies depending on the state of the action being performed.

http://www.uwsp.edu

Button Usage

Default Button

The default button is used for submitting forms & other actions.

HTML code for default button:

<button type='button' class='default-btn'>Default</button>

CSS for default button:

.default-btn {

background-color: #9E9E9E;

border-color: #9E9E9E;

border-radius: 10%;

color: #FFFFFF;

box-shadow: 0 1px 2px #fff,

0 -1px 1px #666,

inset 0 -1px 1px rgba(0, 0, 0, 0.5),

inset 0 1px 1px rgba(255, 255, 255, 0.8); }

.default-btn:hover {

background-color: #8E8E8E;

border-color: #8E8E8E; }

Success Button

The success button is used to show when an action was successful.

HTML code for success button:

<button type='button' class='success-btn'>Success</button>

Button Groups

Group buttons using group-btn.

Button groups can be used when you want to select what type of action to perform.

HTML code for button groups:

<button type="button" class="group-btn">Small</button>

<button type="button" class="group-btn">Medium</button>

<button type="button" class="group-btn">Large</button>

CSS for button groups:

.group-btn {

background-color: #9E9E9E;

border-color: #9E9E9E;

border-radius: 3px;

color: #FFFFFF;

box-shadow: 0 1px 2px #fff,

0 -1px 1px #666,

inset 0 -1px 1px rgba(0, 0, 0, 0.5),

inset 0 1px 1px rgba(255, 255, 255, 0.8);

margin-left: -5px;}

.group-btn:hover {

background-color: #8E8E8E;

border-color: #8E8E8E;}

Lists

Lists are put into name-value pairs.

Use list-name and list-value for lists and use .list-info on an <ul>.

  • name
    value

Html code for name/value list:

<ul class="list-info">

<li>

<div class="list-name">name</div>

<div class="list-value">value</div>

</li>

</ul>

CSS for name/value list:

.list-info {

list-style-type:&nbsp;none;}

.list-name {

text-transform: uppercase;

margin-right: 100px;

border-bottom-style: solid;

border-bottom-color: #B6B6B6;}

.list-value {

border-bottom-style: solid;

border-bottom-color: #B6B6B6;}

.list-value, .list-name {

display:inline;}

Tables

Tables should take most of the viewport and be left-aligned. Use the .table-timesheet class on <table> to create the timesheet.

Job Code Hours Duties Performed
11 8 Loading shipments onto trucks
04 3.75 Cleaning out stockroom

Html code for tables:

<table class="table-timesheet">

<tr>

<th>Job Code</th>

<th>Hours</th>

<th>Duties Performed</th>

</tr>

<tr>

<td>11</td>

<td>8</td>

<td>Loading shipments onto trucks</td>

</tr>

<tr>

<td>04</td>

<td>3.75</td>

<td>Cleaning out stockroom</td>

</tr>

</table>

CSS for tables:

table.table-timesheet {

border-collapse: collapse;

width: 100%;}

th,td {

text-align: left;

padding: 8px;}

tr:nth-child(even){background-color: #f2f2f2}

th {

background-color: #4CAF50;

color: #FFFFFF;}

Dialogs

Dialogs will have a trigger, content area, and dialog actions within the content area. While the dialog is present, the trigger will turn into a warning button then when the action is completed, it will turn into a success button. This comes in handy when the user is submitting a timesheet.

Are you sure you want to continue?

Submitting this form will be final and you cannot revert back to make changes.

 

Html code for dialogs:

<div class="dialog">

<h4 class="dialog-header">Are you sure you want to continue?</h4>

<p>Submitting this form will be final and you cannot revert back to make changes.</p>

<button type="button" class="default-btn">Submit</button>&nbsp;<button type="button" class="default-btn">Cancel</button>

</div>

CSS for dialogs:

.dialog {

background-color: #FFFFFF;

border-radius: 6px;

height: 140px;

width: 280px;}

Navigation Menus

Links will be displayed in a horizontal band. Active link will have a gray underline.

Link Link Another link Link

Html code for navigation menu:

<nav class="navmenu">

<a class="nav-link active" href="#">Link</a>

<a class="nav-link" href="#">Link</a>

<a class="nav-link" href="#">Another link</a>

<a class="nav-link" href="#">Link</a>

</nav>

CSS for navigation menu:

.nav-link {

    margin-right: 40px;}

.active {

    border-bottom-style: solid;

    border-bottom-color: #B6B6B6;}

    Text Inputs

    When it comes to inputs, simplicity is key. The user should be focused on what they are typing. On the Graham-McCroy style guide we have a light grey outline to show the user where the border is along with a helpful placeholder.



    Html code for navigation menu:
