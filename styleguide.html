Graham McCroy Style Guide

This guide & code repository is used for designing the human resource management system at various organizations.

General Purpose

This style guide was designed to make the work for thousands of employees at various organizations painless. It was designed to create a sort of conformity within the site which in return, provides stability. By making all the designs relatively barebones and the color, font pop out, we make tedious work painless and human errors easy to see. This style guide was designed mainly with human resource management in mind as well as making fully functional web applications. Creating a template for developers to start from, is a lot easier than starting with nothing. Not only that, but maintaining all of the website's elements in a clean, consistent code, benefits both the developer and the user.

Introduction to the Grid System

This grid system was designed to be responsive above everything else. Most sites today use the 12-column grid system. This guide is no different. This grid system is mobile first and scales up to 12 columns as the viewport size increases. People use all types of devices to access content. If users have trouble accessing content on either their mobile device, tablet, or desktop, more than likely, they will never visit the site again. If someone who uses a site designed using this style guide and cannot access content the way they wanted to; and they have to access that content day to day, it would make that part of their job miserable to perform.


This grid system was generated by responsivegridsystem.com/

Typograghy

We achieve styling through opt in classes and do not style elements globally. This is to reduced the number of style collisions across projects. Elements can inherit styles from .copy--body or otherwise applied directly.

Typographic elements do not come with any margins attached to them in order to make them more flexible in where they can be used.

They get margins when they are inside a wrapper with a class of context--content. We may need to work on the naming but the principle is solid.
